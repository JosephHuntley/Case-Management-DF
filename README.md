# HTTPS (HIGHLY RECOMMENDED)

In order to use HTTPS you must complete the following steps.

## 1. Add the server to your hosts file

If running standalone (server and browser on the same machine):

`127.0.0.1  case-df.local`

If accessing over LAN (multi-investigator mode), use the server's LAN IP instead:

`192.168.1.50  case-df.local`

## 2. Generate the certificate

From the project root:
```
mkcert -install
mkcert case-df.local
```
This creates `case-df.local.pem` and `case-df.local-key.pem` in the current directory.

## 3. Set environment variables

```
USE_HTTPS=true
SSL_KEYFILE={path_to_keyfile}
SSL_CERTFILE={path_to_certfile}
```

## 4. On any other system that will connect to this server

Each machine needs to trust the certificate and resolve the hostname:

1. Add the same hosts file entry as above, pointing to the **server's** LAN IP (not `127.0.0.1`).
2. Copy the root CA from the server (found via `mkcert -CAROOT` on the server machine) to this machine, then run `mkcert -install` locally to trust it.

Without step 2, browsers on other machines will show a certificate warning even if the hostname resolves correctly.

# Environment Variables

Create a `.env` file in the project root with the following variables:

```
SECRET_KEY=your-secret-key-here
ENV=development
USE_HTTPS=false
PORT=8000
SSL_KEYFILE=./certs/case-df.local-key.pem
SSL_CERTFILE=./certs/case-df.local.pem
```

## Variable reference

| Variable | Required | Default | Description |
|---|---|---|---|
| `SECRET_KEY` | Yes | — | Secret key used to sign JWT access/refresh tokens. Must be a long, random string.  |
| `ENV` | No | `development` | Application environment. `development` enables database seeding on startup. Set to `production` for real deployments to disable seeding. |
| `USE_HTTPS` | No | `false` | Whether to run the server over HTTPS. Set to `true` for LAN or multi-investigator deployments. See [HTTPS setup](#https-highly-recommended) above. |
| `PORT` | No | `8000` | Port the server listens on. Use `8443` (or similar) when `USE_HTTPS=true`, to avoid conflicting with the plain-HTTP default. |
| `SSL_KEYFILE` | Only if `USE_HTTPS=true` | — | Path to the private key file generated by `mkcert`. |
| `SSL_CERTFILE` | Only if `USE_HTTPS=true` | — | Path to the certificate file generated by `mkcert`. |

## Generating a `SECRET_KEY`

You can generate a secure random key with:
```bash
python -c "import secrets; print(secrets.token_hex(32))"
```
